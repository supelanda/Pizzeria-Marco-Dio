Setup Supabase — istruzioni SQL (solo sviluppatore)

Esegui questo SQL nel pannello Supabase → SQL Editor per creare la tabella e configurare RLS.

-- Tabella prenotazioni
create table prenotazioni (
  id uuid default gen_random_uuid() primary key,
  created_at timestamptz default now(),
  nome text not null,
  telefono text not null,
  data date not null,
  ora text not null,
  persone integer not null,
  note text,
  stato text default 'in_attesa'
    check (stato in ('in_attesa','confermata','cancellata'))
);

-- Abilita RLS
alter table prenotazioni enable row level security;

-- Chiunque può inserire (form pubblico)
create policy "insert_public" on prenotazioni
  for insert to anon with check (true);

-- Solo utenti autenticati possono leggere/modificare
create policy "all_authenticated" on prenotazioni
  for all to authenticated using (true);

-- Opzionale: indice per data
create index idx_prenotazioni_data on prenotazioni(data);

Per la pagina admin usa la service_role key (non anon key) — può bypassare RLS. Trovala in Supabase → Project Settings → API → service_role.
